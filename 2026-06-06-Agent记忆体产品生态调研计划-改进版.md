# AI Agent 记忆体产品生态调研计划（v10.1 改进版）

**计划版本**：v10.1（在 v10.0 基础上补充产品覆盖、对标维度、决策框架）
**原始版本**：v10.0（2026 年 6 月 1 日）

> 📌 标注为本次改进新增内容。

## 一、调研背景

（保持原 v10.0 内容不变）

📌 **补充背景**：当前赛道正分裂为两条技术路线——**认知深度路线**（以 Hindsight 为代表，追求事实/信念分离、观点可演化、置信度量化）与**记忆广度路线**（以 MIRIX 为代表，追求六种记忆类型的完备覆盖、多 Agent 协同、多模态支持）。两条路线在基准表现上均已达到 SOTA 水平（LongMemEval 91.4% vs LoCoMo 85.38%），但设计哲学差异巨大，直接导致选型场景完全不同。本调研需同时覆盖两条路线的代表性产品。

---

## 二、调研目标

（保持原 v10.0 第 1-4 项目标不变）

📌 **新增目标 5**：（接原目标 4 之后）
5. 覆盖认知深度与记忆广度两条技术路线的代表性产品，建立跨路线的统一评价框架；

📌 **新增目标 6**：
6. 引入独立基准测试（LongMemEval、LoCoMo、ScreenshotVQA、AMB）作为产品能力的可量化验证手段，解决纯定性对比的不足。

---

## 三、调研原则

（保持原 v10.0 第 1-4 条不变）

📌 **新增原则 5**：**基准可验证原则**——对不同产品的能力描述必须引用可复现的基准测试数据（如 LongMemEval、LoCoMo），无法提供基准数据的产品需标注"未公开基准测试"。

📌 **新增原则 6**：**路线覆盖完整性原则**——调研产品池必须以表格形式标注各产品所属的技术路线（认知深度 / 记忆广度 / 传统向量检索），确保两条路线均有足够的代表产品纳入对比。

## 四、层级划分标准（按产品原生能力）

（保持原 v10.0 的三层标准不变）

📌 **补充说明**：原 v10.0 第三层代表列举了 Qdrant、pgvector、Neo4j、Milvus，完整第三层产品池建议扩充至（至少应覆盖）：

| 类别 | 代表产品 |
|------|---------|
| 向量数据库 | Qdrant, Milvus, Weaviate, Pinecone, Chroma, pgvector, Redis, Elasticsearch |
| 图数据库 | Neo4j, ArangoDB, TigerGraph, JanusGraph |
| 混合引擎 | SurrealDB, EdgeDB |

📌 **第二层产品池扩充建议**（v10.0 仅列 Mem0 SDK、Graphiti、LangMem、PowerMem，建议至少补入以下产品并标注路线）：

| 产品 | 路线 | 必须纳入的理由 |
|------|------|---------------|
| **Hindsight** | 认知深度 | LongMemEval 91.4%（SOTA），LoCoMo 89.61%，唯一区分事实/信念/观点的系统，唯一含置信度+行为参数配置 |
| **Letta** | 认知深度（状态化 Agent） | 层次化记忆（working/recall/archival），支持自我进化，ADEPT 基准 |
| **Cognee** | 记忆广度（图谱驱动） | 从文档和对话中构建知识图谱，支持结构化实体建模 |
| **Mem0 SDK** | 传统向量+图谱 | 社区最大（48K stars），双存储架构（向量+图），快速接入 |
| **Zep** | 传统向量+图谱（时序） | 时间感知知识图谱，LoCoMo 79.09%，企业部署成熟度最高 |
| **LangMem** | 框架嵌入式 | LangChain 原生，LoCoMo 78.05%，适合已有 LangChain 的项目 |
| **MIRIX** | 记忆广度（多 Agent） | 唯一六记忆组件+八 Agent 架构，唯一多模态截图记忆，ScreenshotVQA SOTA，LoCoMo 85.38% |
| **Graphiti** | 时序知识图谱 | 时间感知实体关系建模，处理动态实体关系 |
| **PowerMem** | 传统检索增强 | 轻量级，快速接入 |

> 📌 MIRIX 的分类说明：MIRIX 同时具备第二层（记忆业务逻辑封装）和类第一层（自有跨平台 App）的特征，但其核心能力在记忆组织层（六组件体系），建议按**第二层**定级。其自有 App 作为应用案例而非独立商业产品。

### 5.2 第二层：记忆中间件 — 核心能力

（保持原 v10.0 能力域不变）

📌 **新增能力域（认知质量维度）**——这是当前行业的最短板，也是区分"检索好"和"真正理解记忆"的关键评估项：

| 能力域 | 具体能力 | 说明 | 当前行业覆盖度 |
|--------|---------|------|:--:|
| **事实/信念分离** | 区分客观事实（"Alice 在 Google 工作"）与主观判断（"Alice 适合做前端"） | 是否将两类记忆分网存储、分别检索 | 仅 Hindsight |
| **观点可演化** | 随着新证据更新已有的主观判断 | 是否支持带时间戳的信念链，是否可追溯变化历史 | 仅 Hindsight |
| **置信度量化** | 为每条记忆分配置信度分数 | 分数是否由系统自动计算（非手动标注），是否影响检索权重 | 仅 Hindsight |
| **行为可配置** | 可调节记忆检索时的怀疑程度、字面解释程度、同理心 | 是否为系统级配置，是否可针对不同场景调整 | 仅 Hindsight |
| **多模态记忆** | 支持图像、截图、文档作为记忆输入 | 是否支持非文本记忆的提取、索引和检索 | 仅 MIRIX |
| **隐私加密** | 敏感记忆的端到端加密存储 | 加密粒度（字段/条目/库级别），密钥管理方式 | MIRIX（知识保险库），其他待调研 |

📌 **新增能力域（基准测试对标）**——用可复现的第三方基准验证产品能力声明：

| 基准 | 测试内容 | 当前 SOTA | 需对比的产品 |
|------|---------|-----------|-------------|
| **LongMemEval** | 长期对话记忆的准确召回 | Hindsight 91.4%（大主干），83.6%（20B 开源） | Hindsight, Mem0, Zep, LangMem, Letta |
| **LoCoMo** | 长对话中的记忆一致性 | Hindsight 89.61%，MIRIX 85.38%，全上下文理论上限 87.52% | Hindsight, MIRIX, Mem0（62.47%）, Zep（79.09%）, LangMem（78.05%） |
| **ScreenshotVQA** | 多模态截图的记忆与问答 | MIRIX 59.5%（RAG 基线 44.1%，Gemini 长上下文 11.7%） | 仅 MIRIX（其他产品不支持多模态） |
| **Agent Memory Benchmark (AMB)** | 综合记忆质量（准确率、召回、延迟、成本四维度） | 待确认（Vectorize 2026.03 发布） | 所有第二层产品 |

## 六、📌 新增：两条技术路线的对比框架

调研需在汇总阶段输出一份路线对比矩阵，帮助后续选型决策：

| 对比维度 | 认知深度路线（Hindsight 为代表） | 记忆广度路线（MIRIX 为代表） | 传统路线（Mem0/Zep 为代表） |
|---------|-------------------------------|----------------------------|----------------------------|
| **核心命题** | "让 agent 真正理解自己记住了什么" | "让 agent 拥有类人的完整记忆体系" | "让 agent 能记住和检索信息" |
| **记忆组织** | 4 网络（世界/经验/观点/观察），按认知类型分 | 6 组件（核心/情景/语义/程序/资源/保险库），按记忆类型分 | 向量索引 + 实体图（可选） |
| **事实/信念分离** | ✅ 核心特征 | ❌ | ❌ |
| **多模态** | ❌ | ✅ 截图记忆 SOTA | ❌ 或有限 |
| **Agent 架构** | 2 模块（Tempr+Cara），单 Agent | 8 Agent 协同 | 单服务架构 |
| **行为可配置** | ✅ 怀疑度/字面度/同理心 | ❌ | ❌ |
| **框架集成** | ✅ 主流框架全覆盖 | ❌ 无成熟框架 SDK | ✅ 主流框架覆盖 |
| **适用场景** | 编程 Agent、知识工作、需要信念演化的场景 | 个人助理、屏幕活动追踪、多模态场景 | 对话机器人、客服、简单偏好记忆 |
| **典型用户** | 以 Agent 为主的工程团队 | 以用户为主的个人助理场景 | 以对话为主的轻量场景 |

## 七、📌 新增：行业生态缺口与待解决命题

原 v10.0 第四项目标提到"梳理行业生态缺口"，建议在本节补充当前可识别的主要缺口：

1. **无统一基准**：LongMemEval 和 LoCoMo 只测对话记忆，不测任务改进效果。需要"记忆是否让 Agent 在下一次任务中表现更好"的基准（目前 AMB 是这个方向但尚未普及）。
2. **认知深度产品仅 Hindsight 一家**：事实/信念分离、观点可演化、置信度量化——这些能力在当前市场上是独家的。如果 Hindsight 的母公司 Vectorize 出现商业风险，整个认知深度路线缺少备份选项。
3. **多模态记忆产品仅 MIRIX 一家**：且 MIRIX 无商业实体支撑，长期维护风险较高。
4. **中文记忆提取**：所有主流产品的中文记忆能力均未经过独立的基准测试验证。
5. **框架集成碎片化**：Hindsight 集成最广但认知深度路线只有它一家；MIRIX 完全没有框架 SDK；市场上缺乏"兼具认知深度和框架广泛集成"的产品。
6. **国内产品缺位**：第二层（记忆中间件）几乎无国内自研产品，第三层（存储引擎）和第一层（MaaS）有国内厂商但能力对标国际仍有差距。

## 八、📌 补充：长效迭代机制的具体操作建议

原 v10.0 提到"搭建长效迭代调研体系"，建议增加以下具体机制：

1. **基准追踪表**：维护一份独立于产品清单的基准分数追踪表（LongMemEval / LoCoMo / ScreenshotVQA / AMB），每季度更新一次，记录各产品的最新基准成绩。
2. **产品动态日志**：每个待调研产品分配一条"动态日志"记录，记录产品版本更新、架构变更、融资/收购、社区里程碑。
3. **选型速查卡**：每季度从调研数据库中抽取一份"选型速查卡"（一页内包含：产品名、技术路线、核心指标、对标场景、风险提示），为选型决策提供轻量化入口。
4. **调研责任人轮换**：建议 2 人以上交叉维护，避免单点知识依赖。
