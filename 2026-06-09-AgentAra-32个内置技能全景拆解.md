## AgentAra 是什么

**AgentAra**（[MagicCube/agentara](https://github.com/MagicCube/agentara)，⭐456 / 🔀53 / MIT / 最近更新 2026-06-09）是一个本地运行的 AI Agent 平台，后端用 Bun + TypeScript + Hono 构建，前端 React 19 + Vite 7 仪表盘。它的核心思路是：**驱动 Claude Code 或 OpenAI Codex 会话，通过 Skills 系统赋予 Agent 特定领域的专业能力。**

技术栈一览：

| 层 | 选型 |
|---|---|
| 运行时 | Bun |
| 语言 | TypeScript |
| API | Hono |
| 数据库 | SQLite（Bun 内置）+ Drizzle ORM |
| 任务队列 | Bunqueue |
| 前端 | React 19 + Vite 7 + Tailwind v4 + Shadcn |
| Agent 驱动 | Claude Code / OpenAI Codex |

平台特性包括多通道消息（飞书等 IM）、消息流式更新、任务队列（会话内串行 + 跨会话并发）、Cron 定时任务、会话持久化（JSONL）、Web 仪表盘、文件和图片收发。

但平台本身只是骨架。**真正的能力来自 Skills 系统**——每个 Skill 是一份结构化的 Markdown 指令文件（`SKILL.md`），告诉 Agent 在特定场景下该怎么思考、调用什么工具、遵循什么规范。Skill 不是写死在代码里的功能，而是可插拔的"专业技能包"。

目前内置 **32 个 Skill**。下面按功能域逐一拆解。

## 📹 视频创作全流程（4 个）

AgentAra 把短视频创作拆成了 **4 个串行环节**，每个环节的输出是下一个环节的输入：

### video-plan（视频规划）

把一句话创意变成可执行的分镜脚本。协商总时长（30s/45s/60s/90s），规划每个场景（4-15 秒），选视觉风格（电影散文/手持日记/编辑蒙太奇等 3-5 种方向），输出完整的场景表格：镜头类型、角色动作、服装、场景道具、剪辑节奏。提供视觉参考系统（摄影师/导演/电影风格），输出保存为 `plans/video-plan.md`。

### video-storyboard（视频分镜）

拿 video-plan 的场景表，生成两个文件：**分镜图**（`storyboard/scene-XX.png`，3×3/4×3/4×4 网格随场景时长自适应）和**视频生成提示脚本**（`storyboard/scene-XX.md`，含 0.5s 精度时间码和声音设计）。严格角色一致性——面部、发型、服装、道具逐帧锁定，支持角色设计图作为视觉参考。

### video-character-design（角色设计）

支持两种输入：从参考图提取 / 从零设计。输出 16:9 双行布局的角色表——上行全身四视角转面图（左/前/后/右，占 60%），下行面部特写（占 40%）。专业摄影棚试装照风格，服装材质渲染到纤维级。输出到 `characters/` 目录。

### video-poster-design（视频海报）

从视频计划或角色设计中提取关键元素，生成 3:4 电影海报。提供 3-5 个创意方向（角色主导/环境宽幅/物件象征/运动感/极简预告），字体与故事类型自动匹配。输出到 `posters/` 目录。

**工作流**：`video-plan` → `video-character-design` → `video-storyboard` → `video-poster-design`。每个环节独立可用，也可以串成完整 pipeline。

## 🎨 图像与设计（5 个）

### image-generation（图像生成）

通用图像生成引擎。JSON 结构化提示词，支持多参考图输入（风格/构图引导）、完整相机参数（位置/角度/镜头/光圈/景深）、文字叠加层、负向提示词。由 `scripts/generate.py` 执行，支持 16:9/9:16 等多种比例，失败自动重试 3 次。**这是核心基础设施**——`ppt-generation` 等多个 Skill 依赖它。

### ecom-shot（电商产品图）

分析用户上传的产品照片，识别主体，生成专业摄影级英文提示词。四种场景：纯白底主图、生活场景图、创意氛围图、自定义。充当"创意总监"角色，6 要素提示结构（主体/构图/相机镜头/灯光/风格/技术参数），输出可直接复制到 Midjourney/DALL-E 的提示词。支持 1:1/4:3/3:4/16:9 比例。

### glassify-ui（UI 3D 渲染）

把 UI 截图变成 Cinema4D/Octane 风格的照片级 3D 海报。设计哲学叫"舍得"——简化到 4-8 层深度层，一个英雄层作为视觉锚点。微元素系统（按钮、开关、药丸芯片等作为独立玻璃砖），完整的材质/环境/灯光/相机规范。输出纯 JSON 渲染规范，磨砂亚克力玻璃材质，8K 分辨率。

### article-poster（文章海报）

输入文章 URL 或文本，输出信息图海报 PNG。极简 token 消耗（JSON 仅 300-600 tokens），HTML/CSS 模板静态渲染 + Playwright 截图。8 种颜色主题，单/双栏布局，三种宽度（1080/1200/1600px），丰富的卡片类型（text/highlight/tags/compare/bullets/callout）。

### frontend-design（前端设计）

生成有独特视觉记忆点的前端代码（HTML/CSS/JS、React、Vue 等）。铁律：**禁用 Inter/Roboto/Arial 等通用字体**，每次必须有大胆的美学方向（极简/复古未来/有机自然/奢华等），支持非对称、重叠、对角线等非常规空间构图，含动效与微交互设计。

## 📝 写作与研究（4 个）

### technical-writing-skill（技术写作）

模块化 5 阶段流程：主题确认 → 深度调研 → 大纲设计 → 内容创作 → 迭代优化。**渐进式加载策略**节省 60%+ 上下文 token——每阶段有独立指南文档、模板和检查清单，不需要一次性把所有指令塞进上下文。文件结构含 `stage-*.md`、`templates/`、`checklists/` 子目录。

### deep-research（深度研究）

系统性网络研究方法论，替代"搜一次就写"。4 阶段：广泛探索 → 深度挖掘 → 多样性验证 → 综合检查。覆盖 6 种信息类型（事实数据/案例/专家观点/趋势/对比/挑战），迭代精化搜索策略，严格的质量门槛检查。限制 20 步研究、最多 3 个表格。

### github-deep-research（GitHub 深度研究）

对任意 GitHub 仓库的 4 轮深度扫描：GitHub API 元数据 → 生态发现 → 深入调查 → 深度分析。输出含执行摘要、时间线、指标分析、Mermaid 图表的完整报告，每条结论标注置信度（高/中/低）。Python 脚本 `github_api.py` 支持 summary/readme/tree/languages/contributors/commits/issues/prs/releases 子命令。信息来源优先级：官方文档 > 技术博客 > 新闻 > 社区讨论。

### consulting-analysis（咨询分析报告）

生成 McKinsey/BCG 级别的专业研究报告。覆盖 7 大分析领域（市场/品牌/消费者/财务/行业/投资/竞争情报），内置 20+ 专业框架（SWOT/PEST/波特五力/STP/BCG 矩阵/RFM/DCF 等）。两阶段工作流：分析框架生成 → 最终报告生成。"视觉锚点 → 数据对比 → 综合分析"写作流，"数据 → 用户心理 → 策略启示"洞察链。GB/T 7714-2015 参考文献格式。

## 📊 数据与效率（4 个）

### data-analysis（数据分析）

**DuckDB 内存分析引擎**驱动。上传 Excel/CSV → 自动检查结构（sheets/列名/类型/行数）→ 任意 SQL 查询（聚合/窗口函数/CTE/子查询）→ 统计摘要（均值/中位数/标准差/百分位）→ 导出 CSV/JSON/Markdown。支持多文件跨表 JOIN，100MB+ 大文件，SHA256 哈希键自动缓存。

### stock（股票行情）

三大市场（A 股/港股/美股）实时行情查询。数据源 stooq.com（T+1 数据），异常检测系统覆盖：大幅波动、异常波动、成交量异常放大/萎缩。红涨绿跌配色（中国惯例），知名公司智能识别（中文名→股票代码）。

### ppt-generation（PPT 生成）

规划结构 → 逐张生成 AI 图像（**链式参考**：每张引用前一张作为参考保持视觉一致）→ 组合 PPTX。8 种视觉风格：glassmorphism / dark-premium / gradient-modern / neo-brutalist / 3d-isometric / editorial / minimal-swiss / keynote。依赖 image-generation skill，默认 16:9 比例。

### lab-interpreter（化验报告解读）

解读医学化验报告（血常规/肝肾功能/甲功/肿瘤标志物等）。**异常优先原则**：先说异常值。渐进式信息披露（异常摘要 → 逐项解读 → 综合分析 → 建议），跨语言支持（报告语言 vs 用户语言双语标注），紧急值预警机制。支持图片/PDF/文本输入，每次会话一次 AI 免责声明。

## 🔔 信息推送与监控（4 个）

### pulse（脉搏简报）

面向 4000+ 人技术社区的**每日科技综合简报**。`prefetch.py` 批量预取 Product Hunt / GitHub Trending / 新闻 / 股市 / 天气数据。4 大股指行情 + 45 天走势图，6 城天气预报（北京/上海/广州/深圳/杭州/南京），播客推荐。设计为 cronjob 触发，中文全角标点输出。

### heartbeat（心跳检测）

主动式情报脉冲——读取 `USER.md` / `SOUL.md` 构建个性化监控清单，执行定向搜索，通过"值得打断"测试才推送。6 大监控类别：投资/项目/职业/技术栈/日程/兴趣。"值得打断"三重测试：新颖性/可行动性/改变认知。48 小时日期硬过滤。不值得打扰时返回 `[SKIPPED]`。

### daily-hunt（每日猎奇）

Product Hunt + GitHub Trending Top 10 的个性化日报。基于用户记忆标注（⭐相关/🔥热门/👀有趣），保持原始排名不变，仅添加 emoji 标记。"精选"板块为用户量身定制洞察。列表格式适配移动端/IM 阅读。周末使用周五数据。

### daily-pollen（花粉播报）

花粉过敏指数查询，数据源：**中国天气网 × 北京同仁医院**。5 级花粉浓度等级，7 日趋势分析（持续很高/逐渐缓解/波动较大等），分级行动指南（必须做/建议做/禁止做），30+ 城市支持。`scripts/fetch_data.py` 异步获取天气 + 花粉数据，支持 cron 定时推送。

## 🛠️ 系统与工具（6 个）

### weather-report（天气报告）

全球天气查询：当前天气 + 3 日预报。中国城市走 weather.com.cn，其他地区走 weather.com。附实用小贴士（带伞/加衣/防晒/适合户外），°C/°F 自适应。

### current-time（当前时间）

极简时间查询，默认 Asia/Shanghai，支持任意时区。一行 Bash 命令搞定。

### amap（高德地图）

直连高德 Web Service API：地理编码/逆地理编码、IP 定位、天气、路径规划、距离测量、POI 搜索。TypeScript 脚本 `bun scripts/amap.ts`，需要 `AMAP_MAPS_API_KEY`。**脚本优先设计**——能用脚本解决的不交给 Agent 推理。

### scheduled-tasks（定时任务）

通过 REST API（`localhost:1984/api/cronjobs`）管理 cronjob。支持独立/上下文两种模式，cron 表达式/固定间隔/延迟执行/一次性任务，自然语言到 schedule 字段的智能映射。

### claude-usage（Claude 用量监控）

从本地 API（`localhost:1984/api/usage/claude`）获取 Claude 用量数据。ASCII 进度条展示（20 字符宽）5 小时滚动会话限额和 7 天滚动周限额，80%/95% 阈值警告。

### vercel-deploy-claimable（Vercel 部署）

**零认证部署**：打包项目 → 自动检测 30+ 种框架（Next.js/Nuxt/SvelteKit/Astro/Express 等）→ 上传部署 → 返回预览 URL + 认领链接（可转移到自己的 Vercel 账户）。排除 node_modules 和 .git，支持 JSON 输出的程序化使用。

## 🧠 个人成长与元工具（3 个）

### bootstrap（新用户引导）

灵感来自电影《Her》的"觉醒"式对话。3-5 轮对话生成 `USER.md`（用户画像）和 `SOUL.md`（AI 身份）。不是采访式问卷——从对话中推断 MBTI 信号，每轮回答后给出人意料的洞察反馈。8 种语言支持。反模式清单：不做客服调查、不问"你的爱好是什么"。

### fix-my-life（修复生活）

3 阶段交互式生活修复会话：**挖掘（EXCAVATE）→ 设计（DESIGN）→ 承诺（COMMIT）**。核心洞察："你没有坏习惯，你有一个需要这些习惯的身份。"每次只问一个问题，直接不废话。灵感来自 Dan Koe，渐进式加载各阶段文档。

### skill-creator（Skill 创建器）

**元工具**——用于创建和迭代优化 Skill 本身。完整开发生命周期：构思 → 草稿 → 测试 → 评估 → 迭代。支持并行 with-skill vs baseline 对比、量化评估（assertions）+ 人工评审、描述优化器（自动优化触发准确率）、盲比较系统（A/B 测试）。可打包为 `.skill` 文件分发。三层渐进加载（元数据/SKILL.md/捆绑资源）。

## 🔍 发现与辅助（2 个）

### find-skills（发现 Skills）

开放生态的 Skill 包管理器。`npx skills find [query]` 交互式搜索，自动安装到 `skills/custom/`。常见类别推荐（Web/测试/DevOps/文档/代码质量/设计），链接到 [skills.sh](https://skills.sh) 浏览。

### web-design-guidelines（Web 设计准则审查）

从 Vercel Labs 实时获取最新 Web 界面设计准则（GitHub raw URL），逐条检查指定文件的合规性。`file:line` 简洁输出格式，支持文件/模式匹配。

## 设计哲学：Skill 作为可插拔的能力包

AgentAra 的 Skills 系统有几个值得注意的设计决策：

**1. Markdown 即能力。** 每个 Skill 是一份 `SKILL.md`，不是代码。这意味着任何人都能创建新 Skill（用 skill-creator），也能从开放生态安装社区 Skill（用 find-skills）。能力的创作门槛被降到了写文档的水平。

**2. 渐进式加载。** 多个 Skill（technical-writing-skill、fix-my-life、skill-creator）采用分阶段加载策略——不是一次性把所有指令塞进上下文，而是按需加载当前阶段的文档。这直接节省 60%+ 的 token 消耗。

**3. 脚本优先。** 能用脚本解决的问题不交给 Agent 推理。amap 直接调 TypeScript 脚本，daily-pollen 用 Python 异步获取数据，stock 直接请求 stooq.com。Agent 负责决策，脚本负责执行——这是降低幻觉和提高可靠性的关键设计。

**4. Skill 间依赖。** image-generation 是核心基础设施，被 ppt-generation 依赖。video-plan 的输出是 video-storyboard 的输入。Skill 不是孤立的，它们形成了一个有向依赖图。

**5. 元工具闭环。** skill-creator + find-skills 构成了 Skill 的生产和分发闭环——你可以用 skill-creator 创建和优化 Skill，用 find-skills 发现和安装社区的 Skill，再用 skill-creator 改进它们。这让 AgentAra 的能力天花板取决于社区，而不是开发者。
